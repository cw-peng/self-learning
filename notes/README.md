# Notes — {{LANGUAGE}} Knowledge Base

Write one concept per file. Explain it in your own words as if teaching a colleague.

## Format

Use the note template: copy `.claude/templates/note-template.md` or run `/journal` and reference the template.

Each note should cover:
- **Core Idea** — one paragraph in your own words
- **Key Details** — bullet points
- **Example** — minimal code in {{LANGUAGE}}
- **When to Use / When NOT to Use** — understanding boundaries
- **Related Concepts** — cross-reference other notes

## Organization

- One `.md` file per concept.
- Group related concepts in subdirectories if needed.
- Keep filenames descriptive: `move-semantics.md`, not `notes1.md`.
