# Documentation

Workspace documentation. Each file covers exactly one topic.

## Documents

| File | Topic | Read When |
|------|-------|-----------|
| `learning-methodology.md` | The learning loop and daily workflow | First time using the workspace |
| `workspace-guide.md` | Setup, configuration, troubleshooting | Setting up or reconfiguring |
| `prompt-guide.md` | How to use the 6 AI prompts | Learning what each prompt does |
| `review-checklist.md` | Code review and learning review criteria | Reviewing code or weekly progress |
| `coding-style.md` | Code conventions and style guide | Writing code |
| `architecture.md` | Design rationale and extension guide | Understanding or modifying the template |
| `config-schema.json` | JSON Schema for config.json validation | Reference (used by VS Code automatically) |

## Principles

- **No duplication.** If information belongs in two places, one place defines it and the other links to it.
- **Keep up to date.** When you change how you learn, update the docs.
- **Question everything.** If a document isn't useful, improve it or remove it.
