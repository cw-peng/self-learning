# Exercises — Practice Problems

Apply what you learn immediately. Theory without practice is entertainment.

## When to Practice

- **During** a `/teacher` session — the prompt designs a practice exercise.
- **After** learning a concept — before moving to the next topic.
- **Before** a quiz — warm up with targeted exercises.

## How to Organize

- One subdirectory per topic or concept.
- Include a `README.md` in each subdirectory describing the exercise.
- Keep solutions separate from exercises (e.g., `exercise/` and `solution/` subdirectories) so you can retry later.

## Example Structure

```
exercises/
├── smart-pointers/
│   ├── README.md          (exercise description)
│   ├── exercise/
│   │   └── main.cpp       (starter code with TODOs)
│   └── solution/
│       └── main.cpp       (your completed solution)
```
